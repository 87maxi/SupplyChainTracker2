# Intruciones para debug de codigo
   - crea un directorio docs si no existe 
   - analiza el codigo implementado en el workspace web y sc especificamente, ignora los otros projecto
   - analisa la implementacion del framework que se esta utilizando en cada parte del projecto
   - revisa exaustivamente los archivos de configuracion, y hace un reporte de las implementaciones 
   - reporta todos los metodos implementados por librerias y contratos
   - hace una descripcion detallada tanto del frontend como de los contratos
   - se muy claro con la definicion de parametros y methodos implementados
   - hacer una descripcion de los metodos, en cada seccion de los projectos
   - analiza los archivos ABI en .json y constata que las funciones de la aplicacion web y del contrato sean consistentes
   - hace una referencia de la implemtacion del codigo, insertando los fracmentos del codigo en el documento, hacindo referencia a cada projecto   
   - analiza la ui/ux, propone mejoras en el flujo y crea un markdown con el analisis detallando bien las funcionalidades propuestas y el flujo que deberia tener el usuario  
   - guarda el reporte en un archivo markdown en el directorio docs, separado por projecto
